Exploring Web Development: HTML, CSS, JavaScript & Tailwind

HTML

(HyperText Markup Language)

HTML is the standard markup language used to create and structure content on web pages. It forms the basic building blocks of any website, defining elements like headings, paragraphs, images, and links through a system of tags. HTML creates the foundation that holds all the content in place, serving as the skeleton of every webpage you visit.

CSS

(Cascading Style Sheets)

CSS is the language that controls the visual presentation and layout of web pages. It determines how HTML elements are displayed on screen, handling everything from colors and fonts to spacing and positioning. CSS brings websites to life visually, enabling beautiful designs, responsive layouts that work on different devices, and smooth animations that enhance user experience.

JavaScript

JavaScript is the programming language that adds interactivity and dynamic behavior to websites. It transforms static web pages into engaging applications by responding to user actions, updating content in real-time, and communicating with servers. JavaScript handles everything from simple form validation to complex single-page applications, making the web experience interactive and responsive.

Tailwind CSS

Tailwind CSS is a modern utility-first CSS framework that streamlines web development. Instead of writing custom CSS, developers use pre-built classes to style their HTML directly. This approach speeds up development while maintaining design consistency and responsiveness across different screen sizes, making it popular for building clean, modern interfaces efficiently.

How to Build a Simple Website

🌐 Step 1: Prepare the Tools

Before you start, make sure you have:
A text editor - where you'll write your code
👉 Examples: Visual Studio Code, Sublime Text, Notepad++, or even Notepad.

A web browser - to view your webpage
👉 Examples: Google Chrome, Edge, Firefox, Safari, etc.

(Optional) A project folder
👉 Create a folder called simple-website to store your files.

🧱 Step 2: Create the Basic HTML Structures

HTML (HyperText Markup Language) is the foundation of every website.
Create a new file and save it as: index.html

🟣 1. Document Structure
The HTML file starts with <!DOCTYPE html> and <html lang="en">, which define the document as HTML5 and specify that the language used is English. This is important because it helps browsers interpret the page correctly and supports accessibility tools such as screen readers.

🟣 2. Head Section
The <head> contains information about the webpage that isn't directly visible on the screen.

  • <meta charset="UTF-8"> allows all characters (letters, numbers, symbols) to display correctly.
  • <meta name="viewport" content="width=device-width, initial-scale=1.0"> makes the website responsive — it adjusts its layout for both desktop and mobile devices.
  • <title>Web Sederhana</title> sets the title of the tab in the browser.
  • <link rel="stylesheet" href="style.css"> connects the HTML with an external CSS file that controls how the website looks (colors, layout, fonts, etc.).

🟣 3. Body Section
Everything inside <body> is what appears visually on the webpage. It's divided into several major sections: navigation bar, profile (beranda), schedule (tabel jadwal), and contact/footer.

🟣 4. Navigation Bar (<nav>)
The <nav> element contains the website's main navigation menu. Inside it, there's a <div class="wrapper"> that centers all the content neatly on the page.

  • The logo is inside <div class="logo">, and uses to make it clickable (even though the link is empty here).
  • The menu is in <div class="menu"> and built using an unordered list (<ul>). Each item (<li>) contains an anchor link (<a>) that directs users to different sections of the same page — “Beranda” and “Kontak.” When you click one of these, the browser scrolls smoothly to the corresponding section marked with that id.

🟣 5. Profile Section (<section id="beranda">)
This section introduces Jennifer and contains her biodata.

  • <section> groups related content together and id="beranda" makes it easy to link to.
  • <img src="jennifer.jpg" alt="Jennifer"> displays Jennifer's picture. The alt text (“Jennifer”) appears if the image doesn't load.
  • <div class="kolom"> holds the textual content, including <h2> as the title (“Biodata Diri”) and several <p> tags that describe her personal information — name, birth date, height, job, and a short introduction paragraph. The <b> tag is used to make the labels (like “Nama” or “Pekerjaan”) bold, helping readers quickly find key details.

🟣 6. Schedule Section (<div class="table-section">)
This part shows Jennifer's daily routine using a table.

  • The section starts with a heading <h3>Jadwal Jennifer</h3> to introduce the table.
  • <table> defines the structure. The first row uses <th> (table header) for column titles — “Waktu” and “Kegiatan.”
  • Each following row (<tr>) uses <td> (table data) to fill in the time and activity. Tables like this make information easy to read and neatly organized into columns and rows.

🟣 7. Contact Section (<section id="kontak">) The final section gives Jennifer's contact and social media information.

  • It starts with <section id="kontak">, which can be reached when the user clicks the “Kontak” link from the navigation bar.
  • Inside it, there's a <div class="footer"> that groups several smaller <div class="footer-section"> elements. Each one contains a heading <h6> and a short description. The first block gives a short description (“Seorang penyanyi dari Korea Selatan”). The second block shows Jennifer's email. The third block includes a link to her Instagram using <a class="tmbl-merah">. The class name “tmbl-merah” is for styling, so the link looks like a button rather than plain text. This whole footer is often placed at the bottom of a webpage as a way to give visitors contact details or external links.

🟣 8. Closing Tags
The document ends with </body> and </html>, which close the main body and the HTML document respectively. This tells the browser that there's no more content to render.

✨ Summary
In short, this HTML file defines a simple personal webpage divided into clear, meaningful sections:

  • The head for configuration,
  • The nav for easy navigation,
  • The profile section for personal info,
  • The schedule table for daily activities, and
  • The footer for contact and social links.
  • Each part is semantically correct, making the page well-structured, easy to read, and ready for styling with CSS.

🎨 Step 3: Add Style with CSS

CSS (Cascading Style Sheets) is used to make your website look better.
Create a new file and save it as: style.css

🟢 1. Basic Setup and Global Styles
The CSS file starts by styling the entire page.

  • The universal selector (*) resets margins and paddings to 0, sets box-sizing: border-box (so width and height include padding and borders), and applies a clean, consistent layout across browsers.
  • body sets a background color, a font family (often sans-serif for modern looks), and line height to make text readable. It also ensures the layout is centered and tidy.
    This creates the foundation — ensuring every element looks balanced before more specific styling begins.

🟢 2. Navigation Bar (nav and .wrapper)
The navigation section (nav) is styled to stay at the top of the page and guide users easily.

  • It typically uses display: flex to align the logo and the menu horizontally.
  • justify-content: space-between spreads them apart, while align-items: center keeps them vertically centered.
  • Background color (like dark gray or themed tone) makes it visually distinct from the rest of the page.
  • Inside it, .wrapper sets a maximum width and horizontal padding so content doesn't stretch too far on wide screens.
    The result: a clean, fixed navigation bar with the logo on the left and menu links on the right.

🟢 3. Logo and Menu (.logo and .menu ul li a)

  • .logo a usually has font-size, font-weight, and color styles to make the site’s name or title noticeable.
  • .menu ul removes the bullet points (list-style: none) and arranges menu items horizontally with display: flex.
  • .menu ul li a adds spacing, removes the default underline (text-decoration: none), and sets color.
    When you hover over the links, :hover effects (like color change or underline) are often added to indicate interactivity. This makes the navigation intuitive and visually appealing.

🟢 4. Profile Section (#beranda and .kolom)
The profile section combines an image with text.

  • The container often uses display: flex or grid to position the image and text side by side.
  • The image (img) is styled with a specific width, rounded corners (border-radius), and sometimes a shadow (box-shadow) to make it stand out.
  • The .kolom div holds text content with padding or margin to create spacing from the image.
  • Headings like >h2> get a larger font size and bold weight to act as section titles.
  • Paragraphs >p> have margin-bottom to separate each line and improve readability.
    Overall, this styling ensures that the biodata looks structured and visually balanced.

🟢 5. Table Section (.table-section and table)
The table is made clean and professional using simple, minimalistic styling.

  • .table-section centers the content and adds margins so it doesn’t touch other sections.
  • table uses border-collapse: collapse to merge borders neatly.
  • th (table headers) get a bold font and background color to distinguish them from the rest of the rows.
  • td (table data) has padding for space and border lines for separation.
  • Sometimes, tr:nth-child(even) adds alternating row colors to improve readability.
    This gives the daily schedule a clean, organized, and easy-to-scan look.

🟢 6. Contact/Footer Section (.footer and .footer-section)
The footer section is styled to look unified and calm, usually at the bottom of the page.

  • .footer may use display: flex or grid to arrange sections (description, email, and social links) evenly.
  • Background color (often dark) contrasts with white or light text for readability.
  • .footer-section gives each block some margin or padding.
  • .tmbl-merah (red button) styles the Instagram link with background-color: red, padding, and border-radius to look like a button. A hover effect (like darkening the red or adding shadow) makes it interactive.
    This part ties the whole page together and provides an elegant ending to the site.

🟢 7. Responsive Design (Optional Section)
If the CSS includes media queries like @media (max-width: 768px), it’s for making the website responsive. These rules adjust the layout on smaller screens — for example:

  • Changing flex layouts to vertical stacks.
  • Reducing font sizes.
  • Centering images and text for mobile friendliness.
    It ensures the website looks good on both phones and computers.

✨ Summary
This CSS gives structure and personality to the HTML page:

  • The nav bar ensures easy navigation.
  • The profile section introduces the person clearly.
  • The table presents data neatly.
  • The footer gives closure and contact access.
  • Global styles and responsive rules make everything consistent and accessible on all devices.
    The combination of HTML and CSS creates a simple yet complete webpage — organized, visually appealing, and easy to navigate.

🌍 Step 4: View or Publish Your Website

After finishing your website, you can display it:
👉 Quick method: double-click index.html to open it in your browser.
👉 Online method: upload your project to GitHub Pages, Netlify, or Vercel.
These are free and perfect for student projects 🎓
